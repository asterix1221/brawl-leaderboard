# Презентация (черновик)

## 1. Проблема
- Разрозненные лидерборды, отсутствие прозрачности пересчёта рейтингов.
- Сложность модерации и борьбы с фейковыми матчами.

## 2. Решение
- Веб-платформа с открытым API, кастомным рейтингом и админ-инструментами валидации.

## 3. Архитектура
- Контейнерная схема: Web UI → API (Laravel) → DB + Redis + Queue.
- Отдельные воркеры для пересчёта рейтинга и инвалидации кеша.

## 4. Ключевые сценарии
- Авторизация и хранение сессии через JWT/refresh токены.
- Просмотр лидерборда с фильтрами и кешированием.
- Подтверждение матча администратором, пересчёт рейтинга в фоне.

## 5. Данные и модель
- Пользователи, сезоны, матчи, результаты, слепки рейтинга.
- ER-диаграмма: см. `docs/diagrams/er-diagram.puml`.

## 6. Риски и меры
- Анти-фрод (флаги, ручная модерация, лимиты).
- Производительность (кеш, очередь, read-replicas).

## 7. План внедрения
- MVP: авторизация, просмотр лидерборда, ручное подтверждение матчей.
- Доработка: автоматические импортёры матчей, метрики, алерты.
