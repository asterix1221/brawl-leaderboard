## Спорные решения и риски
- **Алгоритм рейтинга**: выбран кастомный Elo-подобный пересчёт; в случае смены логики нужно изолировать её в отдельном сервисе/воркере и хранить версии формул для повторяемости расчётов.
- **Анти-фрод**: результаты матчей подтверждаются администратором, планируется автоматическое флагирование резких скачков рейтинга и повторяющихся пар матчей.
- **Производительность**: чтение лидерборда обслуживается из Redis; при промахе кеша API даёт прямой запрос в БД и ставит warm-up в очередь.
